# 06_protocol_and_guideline_sources

## Protocol and Guideline Sources

This file defines how sources are used to support the most structured,
human-facing parts of the repository.

Its purpose is not to turn the repository into a clinical guideline.

Its purpose is to make clear:

- what kinds of sources support protocol-level structure
- what kinds of sources support first-line or foundation-level logic
- what kinds of sources support real-world human-facing boundaries
- what kinds of sources support consensus, restraint, and nonpromotion
- how guideline-level or structured-human-use sources differ from mechanistic,
  biomarker, and preclinical sources

This file exists because protocol-facing claims need a different standard than
biological-interest claims.

A source that supports mechanism is not automatically strong enough to support
structured use.
A source that supports a biomarker class is not automatically strong enough to
support protocol influence.
A source that supports early translational interest is not automatically strong
enough to support foundation status.

Protocol and guideline sources matter because they sit closest to use.

## Core Position

Protocol and guideline sources do not decide the repository automatically.

They support the parts of the repository where structure becomes most
human-relevant.

These sources are especially important when the repository is clarifying:

- what already has strong human grounding
- what belongs in the foundation layer
- what kinds of interventions are first-line rather than speculative
- what kinds of limits or caution are already mature enough to state more
  strongly
- where protocol design should inherit more confidence
- where protocol design must remain restrained despite growing field interest

This means protocol and guideline sources are not just “practical sources.”

They are sources that help determine when the evidence is strong enough to
support human-facing structure rather than only scientific discussion.

## Why This File Matters

By this point, the repository has already resolved several key positions:

- the current foundation layer is exercise, high-quality dietary patterning,
  and sleep and recovery treatment logic
- protocol design must remain function-first, risk-bounded, and
  nonpromotion-capable
- support must remain support
- exploratory importance does not equal protocol readiness
- protocol structure did not erase the evidence gaps; it clarified how to work
  honestly within them

Those are current repository positions. :contentReference[oaicite:1]{index=1}

That means the source question here is not:
what sounds protocol-relevant?

It is:
what kind of source actually supports the protocol weight being assigned?

## What Counts As a Protocol or Guideline Source

A protocol or guideline source in this repository is a source that helps answer
one of the following questions:

- What already has strong enough human support to deserve structured use?
- What belongs in first-line or foundation logic rather than exploratory logic?
- What limits or standards already exist for validation, use, or escalation?
- What kinds of human-facing boundaries are mature enough to state clearly?
- What kinds of claims are still too weak to enter protocol structure?

A source is not a protocol source merely because it discusses intervention.

It has to clarify what kind of structure the intervention or measurement can
honestly support.

## Main Protocol and Guideline Source Types

### 1. Practice guidelines and first-line treatment sources

These are among the most important sources in the entire protocol-facing layer
of the repository.

They are strongest when they clarify:

- first-line treatment status
- what is already strong enough for structured human use
- where the evidence has matured beyond exploratory interest
- where behavioral or functional interventions already outrank more speculative
  options

These sources are especially important for foundation-layer logic.

### 2. Consensus and standards sources

These sources matter when the issue is not one intervention result, but what
the field currently treats as a meaningful threshold for:

- biomarker validation
- evidence maturity
- trial design
- translational readiness
- minimum standards for use or interpretation

These sources often do some of the heaviest structural work in the repo
because they help support protocol boundaries without pretending certainty
where it does not exist.

### 3. Human outcome and long-horizon evidence sources

These sources matter when the protocol layer is grounding itself in what aging
actually costs in real humans, such as:

- mortality
- morbidity
- healthy aging
- functional maintenance
- resilience
- quality of life
- frailty reduction
- preservation of capacity

These are especially important because protocol logic in this repository is
function-first, not biomarker-first. :contentReference[oaicite:2]{index=2}

### 4. Structured intervention comparison sources

These sources matter when the repo is clarifying why some interventions belong
closer to foundation or support while others remain exploratory.

They are strongest when they help answer:

- which interventions already have stronger human grounding
- which interventions remain mainly mechanistic or translational
- what the current intervention hierarchy actually looks like

These are especially useful for supporting the repo’s existing hierarchy rather
than reopening it casually.

### 5. Protocol-boundary and nonpromotion sources

These sources matter when the protocol section is not making a positive claim
about what to do, but a bounded claim about what should **not** yet be moved
upward.

They are strongest when they support:

- stop conditions
- nonpromotion rules
- why exploratory remains exploratory
- why support should stay bounded
- why function should outrank biomarker enthusiasm
- why risk and burden remain load-bearing

These sources are crucial because structured restraint needs sourcing too.

## What Protocol and Guideline Sources Are Good For

In this repository, these sources are especially useful for:

### 1. Supporting the foundation layer

They help explain why some interventions already deserve strong protocol
weight.

This is especially important for:

- exercise
- dietary pattern quality
- sleep and recovery treatment logic

These interventions are not foundation-level because they are familiar.
They are foundation-level because they have the strongest current human and
functional grounding. :contentReference[oaicite:3]{index=3}

### 2. Supporting human-facing confidence differences

They help distinguish between:

- what is strong enough to structure
- what is strong enough to observe
- what is strong enough only to track
- what is still too weak to promote

This is one of the most important jobs in the whole source section.

### 3. Supporting nonpromotion honestly

A protocol or guideline source can be most useful when it helps answer:

why is this not yet strong enough for a more central role?

That matters because the repository is built not only on promotion logic, but
also on refusal logic.

### 4. Supporting burden and fit realism

These sources often help show that good protocol logic is not only about
biology.

It is also about:

- who can hold the intervention
- what kind of support is realistic
- what counts as sustainable
- what kinds of human outcomes actually matter

This aligns directly with the repo’s fit and sequencing logic.

## What Protocol and Guideline Sources Are Not Good For

These sources are weaker when they are used outside their evidentiary role.

For example, protocol and guideline sources are not automatically the best
sources for:

- proving one molecular mechanism in depth
- proving one biomarker’s technical validity in isolation
- proving one preclinical intervention’s narrow pathway logic
- replacing primary studies when the question is highly specific

These sources are strongest for structure, threshold, and use-or-restraint
logic.

They are not always strongest for fine-grained mechanism.

## How the Protocol Section Uses Sources

The protocol section should rely on sources differently at different levels.

### At the section README level

The protocol README should rely most heavily on:

- consensus and standards sources
- structured human-outcome sources
- intervention comparisons clarifying current human-grounded hierarchy
- sources supporting why foundation-first logic is stronger than novelty-first
  logic

This is where the repo sets its human-facing structure.

### At the individual-file level

Each protocol file may rely most heavily on different source types depending on
its role.

For example:

- `01_foundation_layer.md` should lean heavily on first-line and human-outcome
  sources
- `05_biomarker_use_in_protocols.md` should lean heavily on validation and
  individual-use-limit sources
- `06_function_first_logic.md` should lean heavily on sources supporting
  organismal relevance over molecular prestige
- `07_risk_boundaries.md` and `10_protocol_failure_modes.md` should lean
  heavily on risk, translation, and nonpromotion sources

### At the boundary level

Files dealing with escalation, support, nonpromotion, and transition thresholds
need especially careful sourcing.

These files are strongest when supported by:

- validation standards
- translational caution sources
- function-first evidence
- fit and burden sources
- sources clarifying what is still too weak for promotion

This is where the source burden becomes most precise.

## Protocol and Guideline Source Failure Modes

### Failure Mode 1 | Guideline prestige borrowing

A guideline source is used as if it settles a different question than it
actually does.

### Failure Mode 2 | Human-facing overconfidence

A source that supports one structured use is overread as if it supports a much
broader protocol role.

### Failure Mode 3 | Foundation under-sourcing

The repo assumes the foundation layer is obvious and therefore does not source
it as carefully as more futuristic intervention classes.

This is one of the most important failures to avoid.

### Failure Mode 4 | Consensus overreach

Consensus tone is allowed to substitute for actual evidence clarity.

### Failure Mode 5 | Nonpromotion under-sourcing

The repo sources escalation better than refusal, making its restraint sound more
intuitive than evidence-based.

## What This File Should Eventually Contain

As the repository grows, this file should likely hold or point toward the
sources doing the most structural work for the protocol-facing layer.

Likely source groups include:

- clinical or practice guideline sources
- consensus and standards documents
- human outcome and long-horizon healthy-aging sources
- structured sources supporting foundation-layer interventions
- sources clarifying first-line versus exploratory logic
- sources supporting stop conditions, burden limits, and nonpromotion logic

The point is not to collect every guideline-like source.

The point is to identify which sources actually support the repo’s structured,
human-facing hierarchy.

## Protocol and Guideline Source Questions the Repository Should Ask

Before giving a protocol or guideline source real weight, the repository should
ask:

- What protocol claim is this source actually supporting?
- Is it supporting first-line use, boundary-setting, validation, fit, or
  nonpromotion?
- Is it being used for a wider claim than it actually supports?
- Does it strengthen human-facing structure, or only make the repo sound more
  settled?
- Is a more specific primary, mechanistic, or translational source still
  needed for this point?
- Is this source helping the repo stay clear about what is ready and what is
  not?

If those questions cannot be answered well, the source may not be doing the
right kind of work.
