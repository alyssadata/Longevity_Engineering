# 01_foundational_reviews

## Foundational Reviews

This file defines how foundational reviews are used in the repository.

Its purpose is not to treat review articles as automatic authority.

Its purpose is to show where review-level literature is useful, where it is
limited, and how it should support the structure of the repository without
quietly replacing direct evidence.

A foundational review is useful here when it helps stabilize the map.

It is not useful simply because it is broad.

## Core Position

Foundational reviews matter because this repository is not built from isolated
papers alone.

It is built from:

- recurring patterns across the field
- hallmark-level structure
- biomarker-class limits
- intervention maturity differences
- risk and translation boundaries
- protocol-level constraints

Review literature is often the clearest place where those broader structures
become visible.

But review literature also has limits.

A review can summarize the field without resolving it.
A review can sound more settled than the underlying evidence.
A review can smooth over disagreements that still matter.

This repository therefore treats foundational reviews as structural aids, not
as automatic final answers.

## What Counts As a Foundational Review

A foundational review in this repository is a source that helps clarify the
overall architecture of a domain.

Examples include reviews that help define:

- the hallmark framework
- biomarker classes and their limitations
- intervention-field maturity
- translation barriers
- validation standards
- protocol-relevant constraints
- major open questions

These reviews are important because they help the repository answer:

What kind of problem is this section actually dealing with?

That is different from asking whether one individual intervention works.

## What Foundational Reviews Are Good For

In this repository, foundational reviews are especially useful for:

### 1. Mapping the field

They help show what the major domains are, how they relate, and where the main
patterns or tensions already exist.

### 2. Clarifying evidence maturity

They help distinguish between:

- mechanistic plausibility
- preclinical signal
- early translational interest
- stronger human evidence
- unresolved uncertainty

### 3. Identifying recurring constraints

They are often the clearest place to see repeated cautions around:

- biomarker overread
- translation weakness
- tissue specificity
- population-fit limits
- risk boundaries
- unresolved causality

### 4. Supporting repository structure

They help justify why the repository is organized the way it is, especially
when multiple sections are built around recurring field-wide problems rather
than one study at a time.

## What Foundational Reviews Are Not Good For

This repository does not use foundational reviews as substitutes for direct
evidence when the question is narrower.

They are weaker when the question is:

- did this specific intervention improve human function
- how strong is this exact biomarker for one-person tracking
- what happened in one trial
- how large was one effect
- whether one narrow mechanistic claim is truly supported

Those questions usually require primary sources, not review-level summary.

A review can point toward the answer.
It should not quietly replace it.

## How This Repository Uses Foundational Reviews

Foundational reviews should be used to support sections when the goal is to
clarify:

- structure
- evidence type
- risk pattern
- field-wide constraint
- unresolved controversy
- maturity level of a domain

That means they are especially appropriate in files such as:

- section READMEs
- cross-section synthesis files
- biomarker validation files
- translation and risk files
- open-questions framing files

They are less appropriate when the file is making a narrow claim about one
intervention result, one human outcome, or one specific mechanism claim.

## Why This File Matters

Without clear source logic, foundational reviews can be misused in two
different ways.

### Failure 1 | Underuse

The repository becomes too paper-fragmented and loses the field-level shape of
the problem.

### Failure 2 | Overuse

The repository starts sounding more settled than the underlying evidence
actually is, because review language is carrying more authority than it should.

This file exists to avoid both failures.

## Criteria for Strong Foundational Reviews

A foundational review is stronger in this repository when it does more of the
following:

- clearly defines the domain it is reviewing
- distinguishes stronger evidence from weaker evidence
- names uncertainty rather than smoothing it over
- separates animal, biomarker, and human-outcome logic clearly
- identifies major limitations explicitly
- helps explain why the field is structured the way it is
- remains useful across more than one file or section

A foundational review is weaker when it:

- sounds broad but says little precisely
- collapses evidence maturity into one tone
- overstates confidence
- substitutes consensus tone for actual source clarity
- is used to support claims better grounded in primary studies

## Foundational Review Failure Modes

### Failure Mode 1 | Review-as-proof

A review is used as if it directly proves a narrow claim that actually depends
on primary evidence.

### Failure Mode 2 | Tone inflation

A review sounds mature, so the repository quietly inherits more certainty than
the underlying evidence earns.

### Failure Mode 3 | Consensus smoothing

A review summarizes a field in a way that hides ongoing disagreement that still
matters for this repository.

### Failure Mode 4 | Structural overreach

A review about one part of the field is used as if it governs the whole field.

### Failure Mode 5 | Citation comfort

A section feels more grounded because it has a review citation, even though the
specific claim still lacks the right evidentiary support.

## What This File Should Eventually Contain

As the repository grows, this file should likely hold or point toward the
reviews that do the most structural work for the repo.

Likely categories include:

- hallmark-defining reviews
- biomarker-validation and consortium reviews
- geroscience intervention overviews
- translation and trial-design reviews
- reviews on risk, failure, and evidence maturity
- reviews that clarify why protocol restraint is necessary

The point is not to collect every review.

The point is to identify which reviews actually support the repo’s structure.

## Practical Questions for Using a Foundational Review

Before giving a foundational review real weight in the repository, this section
should ask:

- What structural problem is this review helping clarify?
- Is it being used for the right scope of claim?
- Does it distinguish evidence types clearly enough?
- Is it naming uncertainty honestly?
- Is a primary study needed instead for this specific point?
- Is the review helping the repository think more clearly, or only sound more
  authoritative?

If those questions cannot be answered well, the review may not be doing the
right kind of work.
