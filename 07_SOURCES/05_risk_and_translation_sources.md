# 05_risk_and_translation_sources

## Risk and Translation Sources

This file defines how sources are used to support the risk, restraint, and
translation logic of the repository.

Its purpose is not to collect every cautionary paper.

Its purpose is to make clear:

- what kinds of sources support biological risk claims
- what kinds of sources support translation limits
- what kinds of sources support nonpromotion and boundary logic
- what kinds of sources are needed when the repository says “not ready”
- how evidence for restraint differs from evidence for mere biological
  plausibility

This file exists because risk claims need source discipline too.

A repository can become distorted not only by weak promotion, but also by weak
restraint.

If the repo is going to say that something remains exploratory, risky,
translation-limited, burden-heavy, or not yet promotable, the sources
supporting that judgment need to be clear.

## Core Position

Risk and translation sources do a different job from hallmark, biomarker, or
intervention-enthusiasm sources.

They often support questions such as:

- what still blocks promotion
- what still limits generalization
- where the field is overreading signal
- what kind of failure pattern is already visible
- what kind of evidence is still missing
- what burden or boundary remains unresolved

These sources are often most important when the repository is saying:

- not yet
- not enough
- not broad enough
- not strong enough
- not stable enough
- not interpretable enough
- not promotable on this evidence base

That means risk and translation sourcing is not secondary.
It is one of the main ways the repository protects its structure.

## Why This File Matters

By this point, the repository has already established that:

- the measurement problem is structural
- function takes precedence when biomarkers conflict with it
- exploratory importance does not equal protocol readiness
- the intervention hierarchy is already visible
- protocol design must remain bounded by stop conditions and nonpromotion rules

Those are current repository positions. :contentReference[oaicite:1]{index=1}

That means risk and translation sources now have a very specific job:

to support the repo when it refuses false promotion.

The real source question is not only:
What supports the intervention?

It is also:
What supports the decision **not** to promote it further yet?

## What Counts As a Risk or Translation Source

A risk or translation source in this repository is a source that helps answer
one of the following questions:

- What kind of biological danger is present?
- What kind of interpretive weakness is still unresolved?
- What kind of translation burden blocks broader use?
- What kind of population-fit limit still matters?
- What kind of evidence gap still prevents promotion?
- What kind of failure mode is already visible in the field?
- What kind of boundary should remain load-bearing?

A source is not a risk or translation source merely because it sounds cautious.

It has to clarify what still blocks honest advancement.

## Main Risk and Translation Source Types

### 1. Risk-focused reviews and perspective papers

These are among the most important sources in this part of the repository.

They are often strongest for clarifying:

- recurring risk patterns
- field-wide overclaim tendencies
- where mechanistic promise outruns practical use
- why some interventions remain exploratory
- how different types of risk should be distinguished

These sources are especially useful for the README-level and cross-sectional
files in `04_RISKS_AND_FAILURE_MODES`.

### 2. Validation and consortium sources

These are especially important when the issue is not biological risk alone, but
whether a biomarker, model, or intervention claim is strong enough to support
the next layer of use.

They are often strongest for clarifying:

- what counts as validation
- what counts as translational maturity
- what is still too weak for individual-level use
- what kinds of evidence are missing for stronger protocol influence

These sources help support the repo’s restraint logic by making standards
visible.

### 3. Human translational and feasibility studies

These sources matter because they often show exactly where the translation gap
still is.

They are useful not only when they show signal, but when they show:

- narrow feasibility
- pilot-scale evidence
- disease-adjacent limitations
- burden-heavy procedures
- weak generalizability
- early signal without broad function

These sources are often strongest when the repository is clarifying why
something remains below promotion threshold even though it is no longer purely
preclinical.

### 4. Methodological and interpretation-limit sources

These sources matter when the main issue is not whether an intervention or
biomarker exists, but whether the signal is being overread.

They are often strongest for clarifying:

- cohort-to-individual overreach
- tissue mismatch
- timing mismatch
- model fragility
- uncertainty inflation
- biomarker-function divergence
- false precision

These sources are central to explaining why the repository keeps measurement
and interpretation boundaries sharp.

### 5. Guideline and consensus-style sources

These matter when the field has reached enough human-facing structure that it
can define first-line logic, practice constraints, or a more mature standard of
care in one domain.

They are not useful because they are conservative by default.

They are useful because they help show where the field already has enough human
confidence to be structured differently.

That makes them especially valuable as contrast sources:

they can show where the repo is strongest on grounded use, and where it is
still not there.

## What Risk and Translation Sources Are Good For

In this repository, these sources are especially useful for:

### 1. Supporting nonpromotion honestly

They help answer:
why is this intervention, biomarker, or protocol element still not promotable?

That is one of the most important jobs in the whole source section.

### 2. Clarifying where risk is structural versus temporary

They can help distinguish:

- built-in mechanism risk
- still-weak control or delivery
- disease-context narrowness
- translation fragility
- unresolved burden
- unresolved interpretation

This distinction is critical for the repo’s boundary language.

### 3. Supporting failure-mode logic

These sources help support files dealing with:

- biomarker theater
- exploratory leakage
- burden creep
- disease-signal inflation
- complexity prestige
- function-biomarker divergence
- protocol drift

### 4. Making stop conditions credible

A stop condition is stronger when the repository can point clearly to the kind
of evidence showing why continued promotion would now be dishonest or too weak.

## What Risk and Translation Sources Are Not Good For

These sources are weaker when they are used to do the work of other source
types.

For example, they are not automatically the right sources for:

- proving an intervention mechanism
- proving one biomarker’s predictive power
- defining one hallmark biologically
- substituting for direct human functional evidence

They often support restraint better than mechanism.

That distinction matters.

## How the Risk and Failure Section Uses Sources

The risk section should rely on sources differently at different levels.

### At the section README level

The risk README should rely most heavily on:

- broad risk and translation reviews
- validation and evidence-maturity sources
- sources clarifying why restraint is structurally necessary

This is where the repo sets the risk landscape.

### At the individual-file level

Each risk file should rely most heavily on:

- sources specific to that risk pattern
- translational reviews
- methodological caution sources
- intervention-specific risk papers where needed
- primary human or feasibility studies showing where limitations still remain

This is where the repo needs narrower support for specific boundary claims.

### At the protocol-interface level

When risk logic influences promotion, nonpromotion, or stop conditions, the
source burden gets even higher.

The most important sources then become:

- human functional studies showing what is missing
- translational studies clarifying narrowness or burden
- methodological sources showing why the signal is too weak for escalation
- sources showing unresolved biological risk where relevant
- sources clarifying why caution is not just tone, but evidence-based

## Risk and Translation Source Failure Modes

### Failure Mode 1 | Caution by tone

The repository sounds careful, but the actual source base for restraint is
weak.

### Failure Mode 2 | Promotion-weight sourcing only

The repo sources optimism carefully but treats nonpromotion as if it needs less
evidence.

### Failure Mode 3 | Mechanism overriding restraint

Mechanistic sources are allowed to carry more weight than risk or translation
sources when the question is actually readiness.

### Failure Mode 4 | Weak-boundary sourcing

The repo names a boundary, but the source base does not actually justify how
strong that boundary is.

### Failure Mode 5 | Softened risk through broad reviews alone

A broad review is used to imply caution without enough intervention-specific or
method-specific support for the actual limit being claimed.

## What This File Should Eventually Contain

As the repository grows, this file should likely hold or point toward the
sources doing the most structural work for the risk and translation logic.

Likely source groups include:

- validation and consortium sources
- risk-focused reviews
- translation and trial-design reviews
- methodological caution sources
- human feasibility studies showing narrow signal
- sources clarifying why certain interventions remain exploratory
- sources supporting stop conditions, nonpromotion, and fit limits

The point is not to collect every caution paper.

The point is to identify which sources actually justify the repo’s restraint.

## Risk and Translation Source Questions the Repository Should Ask

Before giving a risk or translation source real weight, the repository should
ask:

- What kind of limit is this source actually clarifying?
- Is it supporting biological risk, translation weakness, fit narrowness,
  interpretation failure, or nonpromotion?
- Is the source being used for a stronger boundary claim than it supports?
- Does it help explain why something is still not ready, or only that it is
  interesting with caveats?
- Is a more specific primary or methodological source needed for this point?
- Is this source helping the repo stay honest, or only sound careful?

If those questions cannot be answered well, the source may not be carrying the
right kind of restraint.
