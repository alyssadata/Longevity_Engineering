# 02_hallmark_sources

## Hallmark Sources

This file defines how sources are used to support the hallmark structure of the
repository.

Its purpose is not to create a long bibliography of aging biology.

Its purpose is to make clear:

- what kinds of sources justify the hallmark map
- what kinds of sources support one hallmark file versus the framework as a
  whole
- how hallmark-level evidence differs from intervention or biomarker evidence
- where the hallmark framework is strong
- where it is still descriptive, contested, or incomplete

This file exists because the hallmark section is the biological base of the
repository.

That base needs source logic of its own.

## Core Position

Hallmark sources are not just “sources about aging.”

They are sources that help define, support, refine, or challenge the biological
architecture used in `01_HALLMARKS`.

That means hallmark sources may do different kinds of work:

- defining the framework itself
- supporting one hallmark as a real and recurring biological domain
- clarifying how hallmarks interact
- identifying whether a hallmark is more upstream, downstream, or amplifying
- showing where the framework is useful and where it may still be too flat

This repository uses hallmark sources to support biological structure, not only
to collect mechanistic details.

## Why This File Matters

The hallmark section already established the main biological map of the
repository.

That section is not built from one paper and not justified by one type of
source.

It depends on:

- framework-defining reviews
- hallmark-specific review literature
- primary studies clarifying individual mechanisms
- cross-hallmark synthesis work
- later updates that refine, expand, or complicate the original framework

This file exists so those roles stay distinct.

Without that distinction, the hallmark section can become either:

- too dependent on the original framework language
or
- too fragmented into isolated biology without a stable map

## What Counts As a Hallmark Source

A hallmark source in this repository is a source that helps answer one of the
following questions:

- What belongs in the aging framework biologically?
- Why is this hallmark considered real and important?
- How does this hallmark interact with others?
- Is this hallmark more causal, amplifying, or downstream?
- What later evidence strengthens or complicates the original framework?
- What biological patterns recur across multiple hallmark files?

A source is not a hallmark source merely because it mentions aging somewhere.

It has to support the map.

## Main Hallmark Source Types

### 1. Framework-defining reviews

These are the sources that define or revise the broad hallmark structure.

They matter because they help stabilize the shared architecture across all
hallmark files.

In this repository, these are among the most important sources in the entire
biology section.

### 2. Hallmark-specific reviews

These are reviews focused on one hallmark domain, such as:

- genomic instability
- telomere attrition
- proteostasis
- mitochondrial dysfunction
- senescence
- dysbiosis

These are often the most useful sources for supporting the internal logic of a
single hallmark file.

### 3. Cross-hallmark synthesis sources

These sources matter because the repository already treats aging as a network of
interacting failures rather than a set of isolated categories.

Cross-hallmark sources help support claims about:

- interaction
- hierarchy
- amplification
- shared burden
- recurring structural patterns

These are especially important for the notes and synthesis layers of the repo.

### 4. Primary mechanistic studies

These matter when the repository is making a narrower or more specific claim
inside a hallmark file.

Examples include:

- a specific cellular mechanism
- a tissue-specific relationship
- a mechanistic interaction between hallmarks
- evidence that a hallmark is reversible, causal, or mainly reflective

Primary studies are not always needed to support the whole map.
But they are often necessary to support finer-grained claims inside it.

## What Hallmark Sources Are Good For

Hallmark sources are especially useful in this repository for:

### 1. Supporting biological inclusion

Why this hallmark belongs in the framework at all.

### 2. Clarifying biological meaning

What the hallmark actually refers to biologically, rather than just naming it.

### 3. Supporting interaction claims

Why one hallmark cannot be read fully in isolation from the others.

### 4. Refining the framework over time

Showing where the hallmark model has been updated, complicated, or extended.

### 5. Supporting cross-section synthesis

Providing the biological basis for later notes such as:

- sensing and regulation may matter as much as damage
- the environment ages with the cells
- cancer entanglement is structural
- epigenetic reprogramming may be unusual relative to one-hallmark
  interventions

## What Hallmark Sources Are Not Good For

Hallmark sources are weaker when they are asked to do work that belongs to
other sections.

For example, hallmark sources are not automatically the right sources for:

- proving an intervention works
- proving a biomarker is individually usable
- proving a protocol change is justified
- settling translational readiness
- deciding risk boundaries in structured human use

Those questions usually require intervention, biomarker, risk, or protocol
sources rather than hallmark-map sources alone.

## How the Hallmark Section Uses Sources

The hallmark section should rely on sources differently at different levels.

### At the section level

The hallmark README and overarching architecture should rely most heavily on:

- framework-defining reviews
- cross-hallmark synthesis reviews
- later updates that refine the original framework

### At the individual-file level

Each hallmark file should rely most heavily on:

- hallmark-specific reviews
- selective primary mechanistic studies where needed
- cross-hallmark sources when interaction claims are important

### At the notes and synthesis level

The notes file and later synthesis files should rely heavily on:

- sources that support recurring patterns across files
- not just sources that describe one hallmark in isolation

## Hallmark Source Failure Modes

### Failure Mode 1 | Original-framework overreliance

The section leans too heavily on the original hallmark framework source and
does not incorporate later refinement, critique, or extension.

### Failure Mode 2 | Mechanism fragmentation

The section becomes a pile of isolated mechanistic facts without retaining the
biological architecture that makes the hallmark model useful.

### Failure Mode 3 | Interaction under-sourcing

The hallmark files describe each category well in isolation but do not support
their interactions strongly enough.

### Failure Mode 4 | Framework inflation

A framework-level review is used as if it automatically proves narrower claims
that really require hallmark-specific or primary evidence.

### Failure Mode 5 | Descriptive-causal collapse

A hallmark is described well, but the sources do not justify whether it is
being treated as driver, amplifier, consequence, or some mixture of these.

## What This File Should Eventually Contain

As the repository grows, this file should likely hold or point toward the
sources doing the most structural work for the hallmark section.

Likely source groups include:

- original hallmark-defining papers
- updated hallmark-framework reviews
- reviews focused on each major hallmark domain
- synthesis sources on hallmark interaction
- sources that clarify cross-hallmark hierarchy, feedback loops, or shared
  biological tensions
- sources that complicate or challenge overly clean hallmark boundaries

The point is not to list everything ever written on aging.

The point is to identify which sources actually support the map used in this
repo.

## Hallmark Source Questions the Repository Should Ask

Before giving a hallmark source real structural weight, the repository should
ask:

- Is this source helping define the framework, or only one mechanism inside it?
- Is this source being used for the right scope of claim?
- Does it clarify interaction across hallmarks, or only one domain?
- Does it help distinguish driver from amplifier, or is it only descriptive?
- Is this source strengthening the biological architecture, or only adding
  detail without structure?
- Is a more specific hallmark review or primary study needed for this point?

If those questions cannot be answered well, the source may not be doing the
right kind of work.
