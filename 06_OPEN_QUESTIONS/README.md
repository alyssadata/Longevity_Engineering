# 06_OPEN_QUESTIONS

This section tracks the questions the repository has not yet resolved.

Its purpose is not to create vagueness.
Its purpose is to keep uncertainty structured.

By this point in the repository, several things are already settled:

- aging is a cross-hallmark systems problem
- the measurement problem is structural
- function takes precedence when biomarkers conflict with it
- the intervention hierarchy is already visible
- protocol design must be foundation-first, risk-bounded, and explicit about
  what is still uncertain

That means open questions should not restart the repository from zero.

They should begin from what has already been earned.

## Core Position

An open question is not simply something interesting to think about.

In this repository, an open question is something that still blocks:

- stronger intervention ranking
- stronger biomarker interpretation
- stronger protocol confidence
- stronger population-fit clarity
- stronger transition from exploratory relevance to structured use

This section exists to keep those unresolved points visible without letting them
quietly disappear into tone, assumption, or premature promotion.

## Why This Section Matters

The repository now has a real structure:

- `01_HALLMARKS` mapped the biology problem
- `02_BIOMARKERS` mapped the measurement problem
- `03_INTERVENTIONS` mapped the candidate intervention field
- `04_RISKS_AND_FAILURE_MODES` mapped where the field fails, misreads, or
  overpromotes
- `05_PROTOCOL_DESIGN` mapped what can and cannot honestly enter structure

That means the remaining unknowns are no longer random.

They are now organized questions.

This section gives them a place to live clearly.

## What Counts As an Open Question

An open question belongs here when it is still unresolved enough to matter for
the architecture of the repository.

That includes questions such as:

- what is still unknown biologically
- what is still too weak to measure clearly
- what intervention evidence is still missing
- what risk boundaries remain unresolved
- what protocol decisions are still blocked by uncertainty
- what transitions are not yet earned

This section should not collect every possible curiosity.

It should collect the questions that still shape what the repository can and
cannot honestly claim.

## What This Section Is For

This section should track open questions across several domains:

### 1. Biology questions

Questions about mechanisms, hierarchy among hallmarks, upstream drivers, and
whether certain interventions really act above or across the hallmark layer.

### 2. Biomarker questions

Questions about which biomarkers are strong enough to matter, which are still
too partial, and how biomarker conflict should be interpreted over time.

### 3. Intervention questions

Questions about which interventions are most likely to mature, which remain too
risky, and what kind of human functional evidence would actually change their
status.

### 4. Risk questions

Questions about which risks are structurally built into the mechanism, which may
soften with better control, and where the field is still most likely to
overclaim.

### 5. Protocol questions

Questions about escalation, sequencing, support logic, nonpromotion, and what
evidence threshold is enough to justify more structured use.

## Governing Constraint

Open questions do not override settled rules.

They sit underneath them.

That means this section does **not** reopen things that are already structurally
resolved in the repository, such as:

- the measurement problem being structural
- function taking precedence when biomarkers conflict with it
- the current foundation layer being exercise, high-quality dietary patterning,
  and sleep and recovery treatment logic
- exploratory interventions remaining below protocol threshold unless the
  evidence actually changes

Those are not open questions here.
Those are current boundaries. 

## Relationship to the Notes Section

`08_NOTES | Emerging Patterns` already contains the cross-section insights that
emerged from building the repo.

This section is different.

The notes file captures patterns that have already emerged.
This section captures the questions that remain unresolved **after** those
patterns were identified.

In other words:

- `08_NOTES` holds what the repository has learned
- `06_OPEN_QUESTIONS` holds what the repository still cannot yet settle

That distinction matters.
