# 04_risk_and_boundary_questions

## Risk and Boundary Questions

This file tracks the risk and boundary questions the repository has not yet
resolved.

The risk section already did important work.

It established that:

- biological risk is part of intervention quality
- measurement and interpretation failure can distort protocol logic
- function-biomarker divergence is a major warning pattern
- population mismatch is a real failure mode
- different interventions have different characteristic risk patterns
- protocol structures can fail even when their components still sound plausible
- stop conditions and nonpromotion rules are necessary, not optional

Those are current repository positions, not open questions here. 

This file is for the risk and boundary questions that remain unresolved
**inside** that structure.

## Core Position

The main risk problem is no longer whether the field has risks.

It does.

The sharper question is:

Which risks are likely to remain structurally built into the work, which may
soften with better evidence or control, and what unresolved boundaries still
block stronger promotion?

This file exists to keep those unresolved boundaries visible.

## What This File Does Not Reopen

This file does **not** reopen the following current repository positions:

- risk is part of intervention quality
- function takes precedence when biomarkers conflict with it
- nonpromotion is part of honesty, not failure
- exploratory importance does not equal protocol readiness
- population mismatch is a real failure mode
- protocol design must remain bounded by explicit stop conditions

Those are current repository boundaries. 

The open questions here begin after those rules.

## Main Open Risk and Boundary Questions

### 1. Which risks are structurally built into the mechanism?

This is one of the most important unresolved questions in the repository.

Some risks may be temporary because:

- measurement is still immature
- delivery is still weak
- human evidence is still narrow
- sequence or fit is still underdeveloped

Other risks may be more structural because they are tied directly to the
intervention logic itself.

Examples may include:

- cancer entanglement in telomerase-related intervention
- identity destabilization in reprogramming
- selectivity limits in senolytics
- complexity drift in combination logic

The repository still needs sharper distinction between:
temporarily unresolved risk versus structurally built-in risk.

### 2. Which boundaries are absolute, and which are context-dependent?

The risk section already distinguished between stronger and weaker boundaries.

What remains open is where the line actually sits.

Questions include:

- when should function decline be treated as an absolute stop condition
- when is biomarker-function divergence a full boundary versus a pause signal
- which biological risks should stop promotion across all populations
- which risks should be interpreted differently in high-burden versus
  high-functioning adults
- which support-layer burdens are acceptable, and which are already too much

The repo still needs a sharper map of absolute versus conditional boundaries.

### 3. How much functional benefit is enough to offset moderate risk?

This is one of the hardest open questions in the whole framework.

The repository already rejects biomarker-only promotion.

But a harder question remains:

If an intervention produces meaningful function benefit, how much risk can it
honestly carry and still remain promotable?

This matters especially for interventions that may remain:

- biologically meaningful
- functionally promising
- but not low-risk

The repo still needs clearer language for what kind of function is strong
enough to justify carrying moderate risk, and what kind is not.

### 4. When does biomarker-function divergence become disqualifying?

The repository already says function decides.

What remains open is the timing threshold.

Questions include:

- how long can biomarker improvement lead before the lack of function becomes
  disqualifying
- when should mixed biomarker and function signal trigger pause rather than
  removal
- when is “the biomarker may be early” still credible
- when does that explanation become a way of protecting weak intervention logic

This is one of the most important open questions for escalation restraint.

### 5. Which interventions are most likely to drift upward quietly?

The risk section already named exploratory leakage and quiet promotion as major
failure patterns.

What remains open is where the repo is most vulnerable to that drift.

Likely high-risk classes include:

- partial epigenetic reprogramming
- senolytics
- rapamycin-centered logic
- NAD+ restoration
- plasma-exchange and circulating-factor interventions
- selected microbiome-directed interventions

The open question is not whether drift is possible.
It is where the repo needs the strongest explicit boundary language.

### 6. How should support-layer burden be bounded?

Support logic is one of the easiest places for quiet drift to appear.

The repository still needs sharper answers to questions such as:

- how much support is too much support
- when does support stop being support and become stack logic
- how much added burden is acceptable if the foundation seems to improve
- when does a support intervention become decorative rather than structurally
  useful

This is a major open boundary question for protocol integrity.

### 7. When does disease-adjacent evidence justify cautious broader use, if ever?

The repository already treats disease-signal inflation as a real failure mode.

What remains open is the threshold question.

Examples include:

- telomere-related evidence in telomere biology disorders
- senolytic evidence in fibrotic or renal disease contexts
- metabolic interventions in prediabetes or burdened systems
- sleep-treatment evidence in diagnosed insomnia
- plasma- or microbiome-related evidence in narrowly defined populations

The open question is:
when does disease-adjacent evidence remain narrow, and when does it begin to
justify broader but still careful extension?

### 8. Which protocol failures should trigger pause fastest?

The risk section named many failure structures.

What remains open is relative urgency.

Questions include:

- which failure signals should cause immediate simplification
- which should trigger pause and reassessment
- which are warning signs but not yet threshold crossings
- which are most dangerous because they still feel like progress while the
  protocol is drifting

This matters because boundaries are only useful if the repo knows how quickly
to act when they are crossed.

### 9. How should combination-specific risk be bounded?

Combination logic is already treated as important and dangerous.

What remains open is how much evidence is enough before combinations are
allowed more protocol weight.

Questions include:

- what kind of additive benefit is enough to justify added burden
- how much attribution ambiguity is acceptable
- when does orthogonal coverage become real, and when is it just stacking
- which combinations deserve stricter boundaries because of hidden antagonism
  or recovery cost

This is still one of the biggest open risk questions in the repo.

### 10. What should count as a true nonpromotion threshold crossing?

The repository already has nonpromotion rules.

What remains open is how strong the threshold language should become in
specific cases.

For example:

- when does “interesting but not enough” become “should not be promoted for a
  long time”
- when does “pause” become “hold”
- when does “hold” become “de-escalate”
- when does “still exploratory” become “structurally unlikely to mature soon”

These distinctions are not fully settled yet.

## Highest-Priority Risk and Boundary Questions for the Repository

Not every risk question carries equal architectural weight.

The current highest-priority questions appear to be:

1. Which risks are structurally built in rather than merely early?
2. Which boundaries are absolute and which are context-dependent?
3. How much functional benefit is enough to offset moderate risk?
4. When does biomarker-function divergence become disqualifying?
5. How should support-layer burden be bounded?
6. What should count as a true nonpromotion threshold crossing?

These are the risk questions most likely to shape future promotion logic.

## What Would Count As Progress on These Questions

This repository should not treat more caution language by itself as progress.

Progress here would mean things such as:

- clearer distinction between structural and reducible risk
- clearer stop versus pause versus hold logic
- better fit-dependent boundary maps
- stronger translation of failure signals into protocol action
- better criteria for when divergence becomes disqualifying
- clearer thresholds for when support or exploratory layers should not rise

That is the kind of progress this file should track.

## Risk and Boundary Failure Modes This File Should Prevent

This file should help prevent several common failures:

### Failure Mode 1 | Endless caution without threshold clarity

Everything is treated as risky, but nothing is clearly bounded.

### Failure Mode 2 | Soft boundary drift

The language of caution remains, but promotion continues anyway.

### Failure Mode 3 | Structural-risk denial

An intervention is treated as merely early when the main boundary may actually
be built into the mechanism.

### Failure Mode 4 | Context confusion

Conditional boundaries are treated as universal, or universal boundaries are
treated as negotiable.

### Failure Mode 5 | Nonpromotion vagueness

The repo says “not yet” without stating what would actually have to change.
