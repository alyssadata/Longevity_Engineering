# 06_transition_threshold_questions

## Transition Threshold Questions

This file tracks the threshold questions that determine whether anything in the
repository is allowed to change status.

By this point, the repository already has:

- named layers
- named risk boundaries
- named failure modes
- named nonpromotion rules
- named open questions across biology, biomarkers, interventions, risk, and
  protocol design

That means the remaining question is not only:
what is still unresolved?

It is also:
what kind of evidence or clarity would actually count as enough to move
something upward?

This file exists to keep that threshold logic explicit.

## Core Position

A transition threshold is the point at which an intervention, biomarker,
support element, or protocol idea has earned a different status in the
repository.

This matters because the repository is no longer missing structure.

What it is missing, in some places, is threshold clarity.

Without threshold clarity, two things happen:

- weak signals get promoted too easily
- real advances get talked around instead of named clearly

This file is meant to prevent both.

## What This File Does Not Reopen

This file does **not** reopen the current repository boundaries already
established elsewhere, including:

- function takes precedence when biomarkers conflict with it
- the current foundation layer is exercise, high-quality dietary patterning,
  and sleep and recovery treatment logic
- exploratory importance does not equal protocol readiness
- protocol design must remain risk-bounded and nonpromotion-capable

Those are current repository boundaries, not threshold questions here. :contentReference[oaicite:1]{index=1}

The open questions here begin after those rules.

## What a Transition Threshold Is

A transition threshold is not just “more evidence.”

It is a stronger change in status.

Examples include:

- exploratory to support-worthy
- support-worthy to more serious protocol relevance
- biomarker-interest to biomarker-usefulness
- biological plausibility to translational credibility
- interesting signal to actual promotion case
- pause-worthy uncertainty to clarified direction

A threshold should therefore be understood as:

a change big enough to justify a new layer of trust.

## Why This File Matters

The repository already has a clear hierarchy.

But a hierarchy is only stable if there are real criteria for movement between
levels.

Otherwise:

- “not yet” becomes indefinite vagueness
- “promising” becomes quiet promotion
- “exploratory” becomes a tone rather than a category
- “support” becomes a catch-all
- “function-first” stays true in principle but weak in practice

This file exists so the repository can say not only what is currently true,
but what would have to change before something else becomes true.

## Main Transition Threshold Questions

### 1. What is enough to move an intervention from exploratory to support-worthy?

This is one of the most important threshold questions in the repo.

An intervention should not leave exploratory status because it becomes more
talked about, more elegant, or more biomarker-active.

The open threshold question is:

What kind of evidence would show that an intervention is no longer mainly a
research-facing idea, but can actually strengthen the foundation in a bounded,
usable way?

Likely threshold components may include:

- clearer human functional relevance
- lower interpretation burden
- manageable risk
- clearer bottleneck logic
- enough fit clarity to justify bounded use

But the exact threshold remains open.

### 2. What is enough to move an intervention from support-worthy to more serious protocol relevance?

Some interventions may become legitimate support before they become strong
enough to shape more central protocol structure.

The threshold question is:

What kind of gain would justify stronger protocol weight instead of merely
bounded support?

Likely components may include:

- clearer function improvement
- clearer resilience or recovery benefit
- stronger reproducibility across fit-relevant contexts
- lower burden relative to gain
- stronger justification for wider use

This threshold is still unresolved.

### 3. What is enough to move a biomarker from interesting to protocol-relevant?

The repository already rejects biomarker sovereignty.

What remains open is the threshold for when a biomarker becomes strong enough
to influence protocol decisions more meaningfully.

The question is not:
is the biomarker real?

It is:
is it strong enough, stable enough, and function-aligned enough to matter in
protocol logic?

Likely components may include:

- better individual-level interpretability
- better reproducibility
- clearer response-tracking value
- lower context distortion
- stronger alignment with functional change

The exact threshold remains open.

### 4. What counts as enough functional improvement to justify escalation?

This question appears across multiple files because it is one of the most
important unresolved thresholds in the whole repository.

The repository already knows function governs escalation.

What it does not yet fully know is:

- how much improvement is enough
- how durable it must be
- whether preservation counts differently from gain
- how to compare broad modest gain versus narrow strong gain
- how to weigh recovery, resilience, and raw performance against each other

This remains a major threshold question.

### 5. When does biomarker-function divergence become promotion-blocking rather than merely cautionary?

The repository already treats divergence as serious.

The unresolved threshold question is:

When does mixed signal move from:
interesting ambiguity

to:
strong enough reason not to promote?

This includes questions such as:

- how long biomarker-first lead time is acceptable
- when “the organism may lag” remains credible
- when the lack of functional confirmation becomes disqualifying
- whether different intervention classes should have different patience windows

### 6. What kind of evidence is enough to cross from disease-adjacent relevance to broader relevance?

Some interventions currently have their strongest logic in narrower contexts.

The open threshold question is:

What would have to happen before the repository would allow a disease-adjacent
signal to support broader healthy-aging use?

Likely components may include:

- clearer transfer across populations
- better function signal outside the original context
- lower fit uncertainty
- lower translation fragility
- stronger evidence that the effect is not only burden-context-specific

This remains unresolved for several intervention classes.

### 7. What kind of evidence is enough to soften a structural risk boundary?

Some risks may remain built into the mechanism.
Others may soften with better control, sequencing, delivery, or fit.

The threshold question is:

What would have to change before the repository would say a previously central
risk is now better bounded?

This is especially important for:

- reprogramming-related control logic
- telomerase-related oncogenic boundary logic
- senolytic selectivity
- combination burden and antagonism
- plasma-procedure translation burden

This threshold remains highly uncertain.

### 8. When does “not yet” become “unlikely soon”?

The repository already has nonpromotion rules.

But one open threshold question remains:

When should the repo move from saying:
not enough yet

to saying:
this is probably not crossing soon without a major evidence change

This matters because not all open questions remain equally open over time.

Some interventions may remain active research priorities while also becoming
less plausible as near-term protocol candidates.

The repo does not yet have a fully explicit threshold for that shift.

### 9. When does simplification become the stronger move than further refinement?

This is a threshold question inside protocol design itself.

A protocol can become more sophisticated without becoming stronger.

The unresolved question is:

At what point does added refinement stop being useful enough to justify its
burden, ambiguity, or complexity?

This threshold matters for:

- support accumulation
- measurement load
- tracking burden
- combination logic
- sequencing creep

### 10. What counts as true threshold crossing versus rhetorical inflation?

This is the meta-threshold question underneath the whole file.

The repository needs a clearer answer to:

How do we tell the difference between:

- a real status change
- a better story
- a better signal
- a stronger trend
- a quiet promotion by tone

This matters because the field is full of things that sound closer to maturity
than they really are.

The repository is supposed to be stricter than that.

## Highest-Priority Transition Threshold Questions

Not every threshold question carries equal structural weight.

The current highest-priority questions appear to be:

1. What is enough to move an intervention from exploratory to support-worthy?
2. What is enough to move a biomarker from interesting to protocol-relevant?
3. What counts as enough functional improvement to justify escalation?
4. When does biomarker-function divergence become promotion-blocking?
5. What kind of evidence is enough to cross from disease-adjacent relevance to
   broader relevance?
6. When does “not yet” become “unlikely soon”?

These are the threshold questions most likely to shape future repository
movement.

## What Would Count As Progress on These Questions

This repository should not treat more optimism as progress.

Progress here would mean things such as:

- sharper promotion criteria
- clearer fit-sensitive transition rules
- better definitions of enough function
- stronger nonpromotion thresholds
- clearer signals for when risk softens and when it does not
- less reliance on tone to imply movement
- more explicit separation between interesting, promising, support-worthy, and
  protocol-relevant

That is the kind of progress this file should track.

## Transition Threshold Failure Modes

This file should help prevent several common failures:

### Failure Mode 1 | Endless “not yet”

Things remain unresolved forever because the repo never states what would
actually count as enough.

### Failure Mode 2 | Quiet threshold lowering

Promotion happens because the language softens, not because the evidence
strengthens.

### Failure Mode 3 | One-threshold-for-all

All interventions are treated as if they should cross upward on the same kind
of evidence despite having very different risk and translation profiles.

### Failure Mode 4 | Function-light promotion

Threshold crossing is treated as achieved before the organismal case is strong
enough.

### Failure Mode 5 | Inflation by framing

The signal gets more sophisticated, and that is quietly mistaken for being more
mature.
