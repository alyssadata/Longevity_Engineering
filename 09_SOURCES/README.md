# 09_SOURCES

This section contains the actual source library for the repository.

Its purpose is not to explain how sources should be used.
That work belongs in `07_SOURCE_LOGIC`.

Its purpose is to hold the real reference base that supports the repository’s
biological map, biomarker logic, intervention rankings, risk boundaries,
protocol structure, and open questions.

This section exists because source logic and source storage are not the same
thing.

The repository now has both.

## Core Position

A source library should do more than collect citations.

It should make the evidence base navigable.

That means sources in this section should be grouped in a way that reflects how
the repository is actually built.

In this repository, that means the source section should help answer questions
such as:

- what sources support the hallmark framework
- what sources support biomarker validation and limits
- what sources support intervention rankings
- what sources support restraint, nonpromotion, and translation limits
- what sources support the strongest human-facing protocol claims

This is therefore a working source library, not a bibliography dump.

## Why This Section Matters

By this point, the repository already has a real architecture:

- `01_HALLMARKS` maps the biology problem
- `02_BIOMARKERS` maps the measurement problem
- `03_INTERVENTIONS` maps the intervention field
- `04_RISKS_AND_FAILURE_MODES` maps the main failure and overpromotion patterns
- `05_PROTOCOL_DESIGN` maps what can and cannot honestly enter structure
- `06_OPEN_QUESTIONS` maps what remains unresolved
- `07_SOURCE_LOGIC` maps how evidence should be weighted and interpreted
- `08_NOTES` maps the strongest cross-section patterns that emerged

That means `09_SOURCES` should now serve as the actual reference base beneath
that structure.

## What This Section Is For

This section should contain:

- actual references
- grouped source lists
- citations organized by evidentiary role
- the main papers, reviews, validation sources, trials, and standards that
  support the repository
- enough structure that a reader can tell where the strongest evidence sits and
  where the evidence base is thinner

This section should not be filled with commentary that belongs elsewhere.

Its job is to hold the source base clearly.

## Source Grouping Logic

The source library should be grouped by repository function rather than by one
flat alphabetical list.

That is because the repository is not built as one undifferentiated argument.

It is built in layers.

The source section should reflect those layers.

## Proposed File Structure

- `01_foundational_reviews.md`
- `02_hallmark_sources.md`
- `03_biomarker_sources.md`
- `04_intervention_sources.md`
- `05_risk_and_translation_sources.md`
- `06_protocol_and_guideline_sources.md`

Each of those files should contain actual citations and references, not only
source logic.

## Minimum Standard for This Section

Every file in this section should aim to do the following:

- group sources clearly
- distinguish source type where useful
- keep the strongest and most structurally important sources visible
- avoid source inflation
- avoid mixing commentary and citation so heavily that the source list becomes
  hard to use
- make it easier to trace which sections of the repository are best supported

## Relationship to 07_SOURCE_LOGIC

`07_SOURCE_LOGIC` explains how the repository thinks about evidence.

`09_SOURCES` holds the actual evidence base.

That distinction should remain clear.

In simple terms:

- `07_SOURCE_LOGIC` = source governance
- `09_SOURCES` = source library

Both matter, but they do different work.

## Starting Position

Current working position of this repository:

- source architecture now exists separately from the source library
- this section should contain actual references
- the source base should be grouped by repo function
- source quantity does not replace source quality
- the strongest sources should be easy to find
- this section should help show where the repository is strongest and where it
  is still more dependent on emerging evidence

## Main Questions This Section Must Answer

- What are the main sources supporting each major section of the repository?
- Which parts of the repository are strongest on human evidence?
- Which parts are strongest on review or preclinical evidence?
- Which sections rely most on validation, restraint, or guideline sources?
- Where is the source base still thinner than the architecture?

## Status

Foundational source-library section.

This section should be treated as the actual citation home of the repository,
so that the evidence base is organized as clearly as the framework it supports.
